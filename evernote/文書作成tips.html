■月報・週報・報告書

# できるだけ短文で簡潔に書く
例）案件1:10件→OK, 案件2:1件→NG
　   案件1は10件完了した。案件2は1件であるため問題がある。

# 報告文は「であった。」と過去形で言い切ること
例）
～を確認した。 
～を修正した。
～を行った。 ～を実行した。
～を達成した。
～を検討した。～を検討することになった。
～する方針になった。
～について○様からご相談頂いた。
～をご報告した。 ～についてご報告した。
結果を○様にご報告した。
作業報告書を提出して完了とする。 報告書を提出して完了となった。
～とし、本案件は完了とした。
以上より、問題なしと判断した。

# 懸念がある場合の文の〆例
～を必要とする。
～を確認する必要がある。 
～を行えていない可能性がある。 
～になっていない可能性がある。
問題がないか見直しをする必要がある。
以下について確認する必要がある。

# 結果や評価は「〇×△」などで一目でわかるようにする
例）評価:〇
　　<評価〇の理由を2行目以降に書く>

# 数字は3桁ごとに「,」つける
例）1,000

# 報告は（）で実施日をつける
例）・aaa様に作業内容をご報告した。(12/3)

# 月報・週報は必ず<項目>・<作業内容>→<結果>のペアで書く
例）
パラメータチューニング
・Kサーバでパラメータチューニングを実行した。
　→前回の打合せで決めたnode数が1,000以上の中間層も探索したが、最も高精度になったモデルの中間層のnode数は800であった。

# 日付間違いないかチェック忘れるな！！！（指摘福田B）

# 月報・週報は作業何も知らない上長がわかるように書く！！！作業の現状がどうなっているかも書く！！！（指摘2回目_20190425）

# 月報・週報は前提となるキーワードは説明してから記載する（いきなり追加データと書かれても読み手はわからない_20190425）

# 月報・週報は未着手の項目でも ・~ の内容を書く（主題だけはだめ_20190524）

# 月報、業務完了報告書などでの略称はどこかに正式名称書くこと（MVのままにしてた）（2017/09/22）
